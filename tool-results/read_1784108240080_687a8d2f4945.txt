   371→[00436] [Heading 2] 17.2 Connection Lifecycle
   372→[00437] [Normal] WebSocket connection lifecycle 8 steps मध्ऍे managed होतो:
   373→[00438] [Normal] •  1. Client opens WebSocket: wss://api.preone.in/ws/chat?token=eyJ...
   374→[00439] [Normal] •  2. Server validates JWT, sends 'connected' event with sessionId
   375→[00440] [Normal] •  3. Client subscribes to channels: { event: 'subscribe', channel: 'room:01HROOM' }
   376→[00441] [Normal] •  4. Server confirms subscription, starts forwarding events for that channel
   377→[00442] [Normal] •  5. Client sends messages, receives messages in real-time
   378→[00443] [Normal] •  6. Heartbeat: client sends ping every 30s, server responds pong (timeout 60s = disconnect)
   379→[00444] [Normal] •  7. Disconnect: client sends 'close' or TCP connection drops — server cleans up subscriptions
   380→[00445] [Normal] •  8. Reconnect: client uses exponential backoff (1s, 2s, 4s, 8s, 16s, max 30s) with jitter

   382→[00447] [Normal] प्रत्येक WebSocket message consistent envelope structure follow करतो — event name, payload, eventId (for dedup), timestamp. ह्या consistency मुळे client-side parsing uniform राहतो.

   384→[00449] [Normal] WebSocket authentication JWT द्वारे — initial connection request मध्ऍे ?token=eyJ... query param, किंवा connected event नंतर auth message. Server validates JWT, returns connected event with sessionId. Invalid/expired token = 401 close code.

   386→[00451] [Normal] Clients subscribe to specific channels (room:01HROOM, class:01HCL, branch:01HBR). Server ने scope validate करावे — teacher फक्त स्वतःच्या classroom च room subscribe करू शकतो, सर्व branches नाही. Subscription scope JWT scope शी consistent असणे mandatory.

   388→[00453] [Normal] Heartbeat ping/pong दर 30 seconds — client ping, server pong. 60 seconds timeout = disconnect. Reconnection exponential backoff (1s, 2s, 4s, 8s, 16s, max 30s) with random jitter — thundering herd problem prevent करण्यासाठी.

   570→[00650] [Normal] प्रत्येक new endpoint साठी governance review checklist:
   571→[00651] [Normal] •  Naming review: resource nouns (plural, kebab-case), no verbs in path, consistent with domain language.
   572→[00652] [Normal] •  Versioning review: every breaking change requires new major version + 12-month overlap plan.
   573→[00653] [Normal] •  Documentation review: OpenAPI spec completeness (all 17 fields per endpoint filled).
   574→[00654] [Normal] •  Security review: auth scheme, input validation, output encoding, rate limit, PII handling.
   575→[00655] [Normal] •  Performance review: latency budget (p95 < 200ms read, < 500ms write), payload size (<10KB read).
   576→[00656] [Normal] •  Backward compat review: additive changes only within major version — no field removal/rename.
   577→[00657] [Normal] •  Quarterly API audit: full spec review, identify unused endpoints (zero traffic in 90 days → deprecation candidates).
   578→[00658] [Heading 2] 27.4 Quarterly API Audit
   579→[00659] [Normal] Quarterly full API audit — (1) Spec drift detection (deployed API vs OpenAPI spec); (2) Usage analysis (zero-traffic endpoints = deprecation candidates); (3) Security review (auth patterns, input validation); (4) Performance review (latency budgets); (5) Documentation review (spec completeness).
