[Heading 1] Executive Summary
[Normal] PreOne Business Rules Catalog (BRC) v1.0 हा PreOne Enterprise Build Roadmap चा तिसरा foundational document आहे — Vision Document आणि Business Process Model (BPM) नंतर. BRC चा primary purpose म्हणजे प्रत्येक business decision ला govern करणारे rules एका centralized, machine-readable, AI-friendly format मध्ये capture करणे, ज्यामुळे Master PRD, DDD models, ERD, API contracts आणि final code सर्व एकाच rule source वर depend करतील.
[Normal] ह्या document मध्ये 176 rules 12 categories मध्ये विभागल्या आहेत: Eligibility, Financial, Operational, Academic, HR, Inventory, Communication, Compliance, Approval Matrix, Notification, Data Governance, आणि Platform/Multi-tenant. प्रत्येक rule ला unique Rule ID (e.g., R-ADM-007) आहे जो PRD acceptance criteria, ADR references, आणि unit tests मध्ये cross-reference केला जाईल. ह्या ID convention मुळे AI assistants ला business context precise मिळतो आणि development team ला ambiguity न पडता rule locate करता येते.
[Normal] BRC हा BPM आणि PRD मधील missing link आहे. BPM फक्त "what happens" define करतो (process flow), पण "why a decision is taken this way" define करत नाही. PRD मध्ये rules feature-by-feature scatter होतात — acceptance criteria मध्ये, edge cases मध्ये, error messages मध्ये. BRC त्या सर्व rules एका जागी consolidate करतो आणि तिथून PRD/DDD/code सर्वत्र reference करतो. ह्यामुळे rule change करायचा असेल तर एका जागी change करावा लागतो — नव्वे ठिकाणी trace करावे लागत नाही.
[Normal] Document ची status आत्ता Draft (Business Review) आहे. प्रत्येक rule ची review business stakeholders (Principal, Owner, Counsellor, Teacher, Accounts, HR) सोबत होईल. Review पूर्ण झाल्यावर status Business Freeze करता येईल आणि नंतर Master PRD Phase सुरू होईल. BRC वर कोणताही change करण्यासाठी formal Change Request (CR) process follow करावा लागेल — version increment, impact analysis, आणि sign-off mandatory.

[Normal] प्रत्येक rule ची ID format R-DOM-NNN आहे, जिथे DOM हा 3-letter domain code आहे आणि NNN हा sequential number आहे. उदा.: R-ADM-007 = Admission domain ची 7वी rule. ही convention stable आहे — एकदा ID assign झाली की ती change होणार नाही, जर rule delete झाली तरी ID retire केली जाते पण reuse होत नाही.

[Heading 2] Change Request Process
[Normal] एकदा BRC Business Freeze झाल्यावर कोणतीही rule change साठी formal CR process mandatory आहे: (1) CR form मध्ये rule ID, current behavior, proposed change, business justification भरावे लागेल; (2) Impact analysis — कोणते PRD sections, ADRs, code modules affected होतील; (3) Product Lead + Eng Lead joint sign-off; (4) Version increment + changelog update; (5) Stakeholder communication.
